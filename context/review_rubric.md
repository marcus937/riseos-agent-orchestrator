# Review Rubric

Reviews should prioritize:

1. Architecture alignment with BB/Jarvis Architect direction.
2. Safety guardrails, including no auto-merge, no production writes, no branch mutation, and no secrets.
3. Service ownership and repo boundaries.
4. Roadmap alignment and production stability.
5. Focused implementation with tests appropriate to risk.

Request changes when work is unsafe, under-tested for the blast radius, misaligned with ownership boundaries, or missing enough context to judge safely.
